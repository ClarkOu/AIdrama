# AI 短剧自动生成工具 — 设计文档

## 一、产品定位

**目标用户：没有任何影视、编程、AI 使用经验的普通人。**

只要你脑子里有一个故事想法，这个工具就能帮你把它变成一部有台词、有画面、有配乐的短剧视频。你不需要懂提示词，不需要剪辑，不需要写剧本格式——只需要用大白话说出你想要什么。

**核心价值主张：**
- 我有个故事想法 → AI 自动生成剧本、分镜、台词
- 我想要某个角色 → 描述外貌或上传一张图，AI 帮你生成角色形象
- 我想改某段剧情 → 直接改文字，AI 自动更新对应的视频片段
- 全程不写一行代码，不调一次参数，不碰一次 prompt

---

## 二、四大模块与依赖关系

```
角色图库（Character Library）
        ↓（角色名 + 参考图 + 固定描述词）
剧情脚本（Script Engine）
        ↓（script_text 可编辑 + Segment 列表 + prompt 可编辑）
视频生成（Video Generator）
        ↓（草稿审核 → 正式生成 → FFmpeg 合成）

AI 生图服务（Image Generator）──► 为角色图库提供图片来源
```

**单向依赖：下游修改不反向触发上游，避免不必要的 API 消耗。**
**AI 生图服务是横向支撑层，独立于主流程，按需调用。**

### 3.1.5 角色图片来源（Image Source）

角色参考图支持三种获取方式，统一归入角色图库，后续流程无差别使用：

```
方式A：文生图（Text to Image）
  用户描述角色外貌 → 调用 Seedream 生图 → 返回图片 → 存入角色图库
  适合：从零创建角色形象

方式B：图生图（Image to Image）
  用户上传参考图（明星/插画/真人）→ 调用 Seedream i2i → 生成统一风格图片
  适合：基于已有形象做风格迁移，提升跨片段一致性

方式C：本地上传
  用户直接上传本地文件 → 存入角色图库
  适合：已有满意的角色图
```

**建议每个角色固定出以下几张：**
- 正面全身（`is_primary: true`，API 调用优先使用）
- 正面半身/头像
- 45° 侧面
- 2~3 种表情（冷漠、愤怒、微笑）

所有图片背景建议纯白，Seedance API 识别更准确。

### 3.3 分集（Episode）

```json
{
  "ep_id": "ep_001",
  "project_id": "proj_001",
  "ep_number": 1,
  "outline": "林战重返公司，被陈总当众羞辱，决意复仇",
  "script_text": "完整剧本原文（供人阅读，不参与API调用）...",
  "script_version": 2,
  "script_locked": false,
  "segments": [ ... ]
}
```

**脚本编辑规则：**

用户不只能改 prompt，也可以直接修改 `script_text`（文字剧本）。修改后系统弹出确认框：

```
"剧本已更改，是否重新分镜？"
├─ 是 → Script Agent 对比 diff，仅对有变动的幕重新生成 Segment（局部 re-gen）
│        已生成视频的 Segment 标记 prompt_dirty = true，等待用户决定是否重生
└─ 否 → 不调 LLM，用户手动修改对应 Segment 的 prompt
```

`script_locked: true` 时禁止编辑（视频全部生成完毕后自动锁定，防止意外触发 LLM）。

### 3.4 分镜（Segment）

```json
{
  "id": "seg_001_01",
  "ep_id": "ep_001",
  "order": 1,
  "act": 1,
  "scene_desc": "豪华办公室，陈总当众羞辱林战",
  "characters": ["林战", "陈总"],
  "dialogue": "陈总冷笑：你以为离开三年，还能回来？",
  "prompt": "25岁黑发男林战低头站在豪华办公室中央，周围同事窃笑。45岁油腻男陈总坐在主座，将文件扔在地上，冷笑说：你以为离开三年，还能回来？林战慢慢弯腰捡起文件，眼神从屈辱转为冷静。竖屏，冷色调，压抑氛围，轻微手持。",
  "prompt_dirty": false,
  "prompt_version": 1,
  "status": "prompt_ready",
  "draft_task_id": null,
  "draft_video_url": null,
  "final_task_id": null,
  "final_video_url": null,
  "retry_count": 0,
  "last_error": null
}
```

**Segment 可编辑内容（前端）：**

| 字段 | 是否可直接编辑 | 说明 |
|------|----------------|------|
| `prompt` | ✅ 直接编辑 | 不消耗 API，立即生效 |
| `dialogue` | ✅ 直接编辑 | 修改对白文本 |
| `scene_desc` | ✅ 直接编辑 | 修改场景描述 |
| `characters[]` | ✅ 直接编辑 | 增减出场角色 |
| `order` | ✅ 拖拽调整 | 不触发任何 API |
| `act` | ❌ 不可直接改 | 由剧本结构控制，需走脚本编辑流程 |

`prompt_dirty = true` 表示该分镜已生成视频但 prompt 随后被修改，需用户决定是否重新生成。

## 四、Segment 状态机

```
[prompt_ready]
      │
      │ 用户批量确认提交
      ▼
[draft_pending] ──► 提交 Seedance Draft API（480p，0.6倍价格）
      │
      ▼
[drafting]
      │
      ├─ 失败 ─► 自动重试（最多3次）
      │          仍失败 ─► 降级 Seedance 1.5 Pro 重试
      │                   仍失败 ─► [failed] 提示用户处理
      │
      ▼
[draft_review] ◄── 用户在此审核草稿视频
      │
      ├─ 用户点"重做" ─► [prompt_ready]（可修改Prompt，不调LLM）
      │
      ├─ 用户点"通过"
      ▼
[final_pending] ──► 提交 Seedance 正式生成（720p/1080p）
      │
      ▼
[generating]
      │
      ▼
[done] ──► 视频已保存，等待合集合成

[skipped] ──► 用户手动跳过，用占位画面填充，不阻塞合成
```

**只有所有 Segment 状态为 `done` 或 `skipped`，才能点击"合成整集"。**

## 五、API 调用精算

| 阶段 | 操作 | API 调用次数 |
|------|------|-------------|
| 建角色库 | 本地上传图片/填写信息 | **0次** |
| 生成全剧结构 | 输出全部集数梗概 | **LLM × 1次** |
| 单集脚本+分镜 | 一次输出剧本+所有Segment+Prompt | **LLM × 1次/集** |
| 用户修改Prompt | 直接编辑本地数据 | **0次** |
| 草稿视频生成 | 并发提交所有Segment | **Seedance Draft × N次（0.6倍价）** |
| 正式视频生成 | 只提交审核通过的Segment | **Seedance × M次（M ≤ N）** |
| 失败重试 | 最多3次/Segment | **Seedance × 最多3次** |
| 合成整集 | FFmpeg本地拼接 | **0次** |

**每集总LLM调用：1次。草稿模式可以在正式生成前过滤掉不合格片段，节省正式生成费用。**

## 六、单集生产 SOP（标准工作流）

```
Step 1 ── 前置检查
  ✓ 角色库已建立（所有出场角色有 name + fixed_desc）
  ✓ 参考图已上传（至少1张/角色）
  ✓ 本集 outline 已确认

Step 2 ── 脚本生成（LLM × 1次）
  输入：outline + world_config + 角色库信息
  输出：script_text + segments[] JSON
  存储：写入数据库，不重复调用

Step 3 ── 分镜审核（0次API）
  用户操作：
    - 阅读每个 Segment 的 scene_desc + dialogue
    - 直接编辑不满意的 prompt
    - 调整分镜顺序（拖拽）
    - 合并/拆分分镜
  完成后点击"批量提交草稿"

Step 4 ── 草稿生成（Seedance Draft × N并发）
  系统自动：
    - 从 characters[] 提取角色，注入对应 ref_images
    - 并发提交（最多10路）
    - 轮询状态，失败自动重试
  用户看到：实时进度条（X/N 完成）

Step 5 ── 草稿审核（0次API）
  用户逐个观看草稿视频
  通过 → 标记 done_draft
  不通过 → 修改 prompt → 回到 Step 4（仅重提该片段）

Step 6 ── 正式生成（Seedance × M并发）
  只提交标记通过的片段
  支持指定输出分辨率（720p / 1080p）

Step 7 ── 集合成（FFmpeg本地，0次API）
  按 order 排序，拼接所有 final_video_url
  输出整集 MP4

```

## 七、LLM System Prompt 模板（阶段3核心）

```
你是一个短剧编剧助手。根据以下信息，一次性生成本集完整脚本和分镜列表。

【项目信息】
剧名：{title}
风格：{tone}
视觉：{visual_style}

【角色库】
{characters.map(c => `${c.name}：${c.fixed_desc}`).join('\n')}

【本集梗概】
{episode.outline}

【分镜规则】
1. 总分镜数 10-14 个（每段对应 10-12 秒视频）
2. 严格按 5 幕结构：
   - 第1幕（开场钩子）：2-3个，情绪压抑
   - 第2幕（冲突升级）：3-4个，持续压抑
   - 第3幕（转折爽点）：2-3个，急速反转
   - 第4幕（连锁反应）：2-3个，爽感持续
   - 第5幕（集尾钩子）：1-2个，悬念留白
3. characters[] 只能使用角色库中的 name，禁止使用代词
4. prompt 要求：
   - 纯叙述句，无任何结构标记（无括号、无标题、无分点）
   - 开头必须包含出场角色的 fixed_desc
   - 不超过 120 字
   - 结尾画面自然衔接下一段

【输出格式】
严格输出 JSON，不附加任何说明文字：
{
  "script_text": "完整剧本原文...",
  "segments": [
    {
      "id": 1,
      "act": 1,
      "scene_desc": "场景简述（中文，供人读）",
      "characters": ["角色名1"],
      "dialogue": "对白原文（无对白填null）",
      "prompt": "完整的视频生成提示词..."
    }
  ]
}
```

## 八、多 Agent 架构

本工具按模块拆分为 1 个编排 Agent + 4 个专项 Agent，通过事件总线通信，各自独立运行、独立测试。

### 8.1 架构总览

```
用户 UI
    │
    ▼
Orchestrator（编排 Agent）
    ├──► Image Agent     ── 负责角色图生成（Seedream t2i / i2i）
    ├──► Script Agent    ── 负责剧本生成 + 脚本编辑 diff 分析
    ├──► Video Agent     ── 负责视频生成（Seedance，含状态机）
    └──► Compose Agent   ── 负责 FFmpeg 合成整集
```

### 8.2 各 Agent 职责

| Agent | 职责 | 输入 | 输出 | 调用外部服务 |
|-------|------|------|------|-------------|
| **Orchestrator** | 协调流程、路由事件、管理任务依赖 | 用户指令 / UI 事件 | 事件分发 | 无 |
| **Image Agent** | 角色图生成与管理 | 角色描述 / 参考图 | 图片文件 + DB条目 | Seedream API |
| **Script Agent** | LLM 生成剧本 + 分镜 / 脚本 diff 局部重生 | outline + 角色库 | script_text + segments[] JSON | LLM（Claude / GPT-4o）|
| **Video Agent** | 提交 Seedance 任务 + 状态轮询 + 重试 | Segment + ref_images | 视频 URL + 状态更新 | Seedance 1.5 Pro API |
| **Compose Agent** | 按 order 拼接视频 | final_video_url 列表 | 整集 MP4 | FFmpeg（本地）|

### 8.3 事件总线设计

所有 Agent 通过事件总线发布/订阅消息，Orchestrator 作为主要订阅者做路由:

```
事件类型                    发布方          订阅方
─────────────────────────────────────────────────
character.image.requested   UI              Orchestrator → Image Agent
character.image.done        Image Agent     Orchestrator → 更新 DB
script.generate.requested   UI              Orchestrator → Script Agent
script.segments.ready       Script Agent    Orchestrator → 写库 + 通知 UI
script.edited               UI              Orchestrator → Script Agent（diff）
segment.draft.submit        UI              Orchestrator → Video Agent（批量）
segment.status.updated      Video Agent     Orchestrator → 推送 UI 进度
segment.all.done            Video Agent     Orchestrator → 解锁合成按钮
compose.requested           UI              Orchestrator → Compose Agent
compose.done                Compose Agent   Orchestrator → 通知 UI 下载
```

### 8.4 脚本编辑 ↔ 分镜联动逻辑

Script Agent 处理脚本修改时，执行以下 diff 分析：

```
用户修改 script_text
        │
        ▼
Script Agent：对比 script_version N 与 N-1 的 diff
        │
        ├─ 判断修改影响范围
        │   ├─ 仅对白/细节 → 轻量编辑
        │   │     → 直接更新对应 Segment.dialogue
        │   │     → prompt_dirty = true（不自动重生，用户决定）
        │   │
        │   └─ 情节/结构改动 → 重量编辑
        │         → 对变动幕重新调 LLM 生成 Segment（局部 re-gen）
        │         → 未变动幕保持原状（视频/prompt 不受影响）
        │
        ▼
更新 script_version += 1，通知 UI 高亮变动的 Segment
```

**原则：最小化 LLM 调用，已通过审核的 Segment 尽量不重新生成。**

## 九、MVP 技术选型

| 层 | 选型 | 原因 |
|----|------|------|
| 后端 | Python + FastAPI | 轻量，方便调用各 SDK |
| 数据库 | SQLite（MVP）/ PostgreSQL（生产） | MVP 无需运维成本 |
| 任务队列 | Celery + Redis | 管理并发视频生成任务 |
| 视频合成 | FFmpeg | 成熟稳定，本地运行 |
| 前端 | Next.js + Tailwind | 快速搭建工作台 UI |
| LLM | Claude 3.5 Sonnet / GPT-4o | JSON 输出稳定 |
| 视频 API | Seedance 1.5 Pro（火山方舟） | 已开放 API，音画同步 |
| 文件存储 | 本地 / 火山引擎 TOS | MVP 用本地，生产迁移 TOS |
